## Defects the dogfood caught (the structural suite missed all three)

This is the cowpath/vibe-walk lesson — structural-green is not works. Three real plugin defects, all fixed, all now covered by regression tests:

**DEFECT 3 (P0, Windows-only, crash).** `relToRoot` (engine/extract.mjs) used a separator-sensitive `startsWith`. `inventory.app.root` is stored posix (`C:/…`) while a `join()`-built catalog path is native (`C:\…`); the mixed-separator test was false, so `relToRoot` returned the absolute path, and a downstream `join(appRoot, that)` doubled the root → `ENOENT`. Crashed the entire auto-write path on the first app with a pre-existing catalog. Every Windows user would hit this immediately. Fix: normalize both sides to posix (case-insensitive for the drive letter) before the prefix test.

**DEFECT 4 (P0, test-breaking).** The test-harness wrapper used `messages={{}}`. next-intl 4.x does NOT echo a missing key — `t('foo')` THROWS `MISSING_MESSAGE`, turning every wrapped test red (61 failures on the first run). Fix: seed the provider with the namespace block the run just extracted, AND add `onError={() => {}}` + `getMessageFallback={({ key }) => key}` so any out-of-namespace key degrades to the key string instead of throwing.

**DEFECT 5 (test-discovery pollution).** The plugin's own backups under `.vibe-lingual/localize/backup/<batch>/` mirror the source tree, including `*.test.tsx` copies. Jest globbed them as live suites and the bare, pre-wrap copies failed (`No intl context`). Fix: the wire jest patch now adds `<rootDir>/.vibe-lingual/` to `testPathIgnorePatterns`.
